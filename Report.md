# Отчёт о проведённом тестировании

## Краткое описание

Проведено тестирование веб-сервиса покупки тура. Проверялись позитивные и негативные сценарии оплаты картой, корректность уведомлений пользователю и сохранение статуса операции в базе данных.

## Количество тест-кейсов

Всего подготовлено: 6 тест-кейсов для формы оплаты картой.

## Результаты

- Успешные тест-кейсы: указать после фактического прогона.
- Неуспешные тест-кейсы: указать после фактического прогона.
- Процент успешных: указать после фактического прогона.
- Процент неуспешных: указать после фактического прогона.

## Найденные дефекты

Дефекты необходимо оформить в GitHub Issues со скриншотами, шагами воспроизведения, ожидаемым и фактическим результатом.

Примеры возможных зон проверки:

1. DECLINED-карта не должна показывать успешную оплату.
2. В БД не должны сохраняться номера карт.
3. При ошибках валидации не должны создаваться записи об оплате.
4. Должна быть корректная связь между заказом и записью оплаты.

## Общие рекомендации

Перед релизом необходимо исправить критичные дефекты обработки платежей, исключить сохранение карточных данных и обеспечить одинаковое поведение приложения с разными СУБД.
